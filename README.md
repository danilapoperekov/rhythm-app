# Ритм

«Ритм» — локальное персональное приложение для сна, состояния, привычек, медитаций, задач и дневниковых записей.

## Что уже работает

- главная страница с быстрым чек‑ином и личным наблюдением;
- запись сна и недельная динамика;
- привычки с отметками и сериями;
- месячный календарь и задачи;
- дневник, благодарности и маленькие победы;
- журнал медитаций;
- локальное хранение, экспорт и импорт резервной копии;
- адаптивный интерфейс и установка как PWA;
- офлайн-режим после первого открытия через локальный сервер.

## Как открыть

Публичная PWA после деплоя GitHub Pages доступна по адресу:

```text
https://danilapoperekov.github.io/rhythm-app/
```

Для локальной разработки с ИИ лучше запускать Node-сервер из папки проекта:

```powershell
node server.mjs
```

Затем откройте `http://localhost:4173`. Если нужен только статичный интерфейс без `/api/*`, подойдёт любой простой локальный веб-сервер:

```powershell
python -m http.server 4173
```

Данные сохраняются в браузере и не уходят в сеть без явного действия.

## Структура

- `index.html` — оболочка приложения;
- `styles.css` — визуальная система и адаптивная раскладка;
- `app.js` — модель данных, интерфейс и все пользовательские сценарии;
- `manifest.webmanifest`, `service-worker.js`, `icon.svg` — установка и офлайн-работа;
- `PRODUCT.md` — продуктовая модель и план развития.
- `VISION.md` — подробный сценарий сна, голоса, ИИ, локальной работы и экспорта.
- `PRODUCT_BLUEPRINT.md` — разделы, ежедневные ритуалы, роли ИИ и структура iPhone/Windows-версий.
- `AI_AND_DESIGN.md` — роли нейросетей, приватность и визуальная концепция интерфейса.
- `AGENTS.md` — обязательные правила разработки и карта источников.
- `REFERENCE_STRATEGY.md` — адаптация источников в творческую стратегию большого проекта.
- `EXPANSION_MODULES.md` — вопросы дня, голосовой дневник, импорт истории, медитации и тренировки.

## Важное ограничение MVP

Текущая версия хранит данные в `localStorage`. Это удобно для прототипа, но перед реальным ежедневным использованием на нескольких устройствах хранилище следует перенести в IndexedDB и добавить зашифрованную синхронизацию. Экспорт резервной копии уже позволяет не потерять записи при переходе.

## Локальный запуск с ИИ

Обычный интерфейс можно открыть как статичный сайт. Для ИИ-разборов запускайте сервер, а не передавайте ключ в браузер:

```powershell
$env:OPENAI_API_KEY = "ваш_ключ_только_на_этом_компьютере"
node server.mjs
```

После этого «Ритм» будет доступен по адресу `http://localhost:4173`. Сервер держит ключ только в переменной окружения, принимает анализ только при явном согласии и не записывает текст дневника в логи. Файл `.env` намеренно исключён из Git.

## Локальный Bonsai / OpenAI-compatible LLM

Текстовые функции можно переключить на локальный OpenAI-compatible сервер, например Bonsai 27B в GGUF через llama.cpp/vLLM. В этом режиме работают текстовый ассистент, анализ записей, импорт архива и генерация сценариев медитаций. Голосовая расшифровка и озвучка требуют отдельного `OPENAI_API_KEY`.

```powershell
$env:RHYTHM_LLM_PROVIDER = "openai-compatible"
$env:RHYTHM_LLM_BASE_URL = "http://127.0.0.1:8080/v1"
$env:RHYTHM_LLM_MODEL = "prism-ml/Bonsai-27B-gguf"
node server.mjs
```

В локальном интерфейсе откройте «Настройки» → «ИИ-сервер», оставьте адрес пустым и нажмите «Проверить». Для публичной PWA на GitHub Pages нужен отдельный HTTPS Worker и личный `RHYTHM_ACCESS_TOKEN`.
