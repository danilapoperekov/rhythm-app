# Защищённый ИИ-сервер для «Ритма»

PWA публикуется на GitHub Pages, поэтому ключ OpenAI нельзя и не нужно добавлять в приложение. Этот проект разворачивает отдельный Node-сервер: он получает запросы только от разрешённого сайта, требует личный токен и хранит ключ OpenAI только в переменных окружения хостинга.

## Развёртывание через Render

1. Войдите в [Render](https://render.com/) и выберите **New → Blueprint**.
2. Подключите репозиторий `danilapoperekov/rhythm-app`. Render найдёт `render.yaml`.
3. В форме переменных окружения укажите:
   - `OPENAI_API_KEY` — ваш ключ OpenAI. Вставляйте его только в поле Render, не в GitHub и не в PWA.
   - `RHYTHM_ACCESS_TOKEN` — придумайте длинный случайный секрет (минимум 24 символа). Это не ключ OpenAI; он защищает личный сервер от чужих запросов.
   - `RHYTHM_ALLOWED_ORIGINS` уже задан как `https://danilapoperekov.github.io`.
4. Нажмите **Apply**. После запуска Render даст HTTPS-адрес вида `https://rhythm-ai.onrender.com`.
5. Откройте PWA → **Настройки** → **ИИ-сервер** и введите этот адрес и `RHYTHM_ACCESS_TOKEN`.

Токен сохраняется только в браузере конкретного устройства. Для iPhone и Windows его нужно добавить отдельно. Если устройство потеряно, замените `RHYTHM_ACCESS_TOKEN` в Render и введите новый на своих устройствах.

## Что защищено

- `OPENAI_API_KEY` никогда не отправляется в браузер и не попадает в репозиторий;
- сервер принимает API-запросы только от разрешённого origin;
- для запросов нужен Bearer-токен;
- действует лимит 60 API-запросов в минуту на IP;
- аудио и дневниковый текст не пишутся этим сервером на диск или в логи;
- без сети и при ошибке аудио/черновики остаются на устройстве.
